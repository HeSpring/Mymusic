# MyMusic - 本地音乐播放器

MyMusic 是一款基于 Electron + React + Vite 构建的现代化本地音乐播放器。它专注于提供流畅的本地音乐播放体验，拥有极低的资源占用和丰富的高级特性。

## ✨ 核心特性

- **🚀 极致性能优化**
  - 异步后台音乐扫描，启动即刻响应，告别白屏。
  - 懒加载机制，分块渲染长列表，大幅降低运行内存占用。
  - 封面图像自动降采样缓存，减少内存和磁盘开销。

- **🎵 强大的播放控制**
  - 支持顺序播放、单曲循环、随机播放。
  - 动态播放队列展示，随时查看和切换待播放曲目。
  - 进度条支持已播放进度高亮显示。
  - 智能播放统计：单曲播放时间超过总时长 2/3 自动记为一次有效播放。

- **🎤 沉浸式桌面歌词**
  - 逐字同步的卡拉OK级歌词效果。
  - 背景完全透明，不遮挡屏幕视线。
  - 支持拖拽定位。
  - 深度自定义：字体颜色、粗体、居中对齐等选项。

- **💾 智能数据持久化**
  - 首次导入本地音乐文件夹后自动保存路径，下次启动后台自动静默扫描更新。
  - 所有用户偏好设置（播放模式、音量、歌词样式等）自动保存，重启不丢失。
  - 强制单实例运行，避免多开导致数据冲突或托盘图标重复。

## 🛠️ 技术栈

- **框架**: [Electron](https://www.electronjs.org/) + [React](https://reactjs.org/) + [Vite](https://vitejs.dev/)
- **语言**: TypeScript
- **样式**: Tailwind CSS
- **音频解析**: `music-metadata`
- **数据存储**: `electron-store`

## 📦 安装与开发指南

### 下载安装包

您可以直接下载打包好的`MyMusic Setup 1.0.0.exe`安装程序进行安装。
安装后，程序会自动关联单实例运行，并在系统托盘中驻留。

### 本地开发运行

1. 安装依赖：

   ```bash
   npm install
   ```

2. 启动开发服务器：

   ```bash
   npm run dev
   ```

3. 构建并打包应用程序（Windows）：

   ```bash
   npm run build:win
   ```

   打包后的安装文件将生成在 `dist` 或 `release` 目录下。

## 📝 常见问题 (FAQ)

- **为什么桌面歌词背景是黑色的？**
  - 请确保您的系统支持硬件加速，Electron 的透明窗口依赖 GPU 渲染。本项目已在 Windows 环境下深度优化了透明窗口的兼容性。
- **如何重置所有设置？**
  - 所有的配置文件保存在 `%AppData%/MyMusic` 目录下。如果需要彻底重置，可以关闭软件后删除该目录。
